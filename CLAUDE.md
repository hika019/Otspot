# git
- mainにコミットするな
- mainにマージはffを使うな

# 方針/実装
- ユーザーの意見に安易に迎合するな
- 実装コストは無視し、やりたいことに対して、ダイレクトに効くものを実装せよ。上っ面の修正で隠したり、それっぽい動きをするようにするな
- 既存のコードを疑え
- テストは可能な限り全分岐を行え。実装を正とするな。複数パターンのデータを用意せよ。
- マジックナンバーは排除しろ。バグの温床。
- テスト1つの実行時間を3分以内にせよ。cargo testが長時間であってはならない。

# バグ対応
- バグ対応は入力から順に疑え
- 既存のコードを信じるな
- 暫定対処ではなく、原因に対してアクションを行え
- マジックナンバーや多数の分岐は悪
- 事実と推論は明確に区別せよ。推論に対してアクションをして事実に昇格させよ

# 実行
- `./scripts/` を使用するなど効率よく進めよ
- 実行には時間がかかるため、cpu700%を上限とし、並列でベンチをするなど効率化を行え
- ベンチの基準はtimeoutが1000sec, eps=1e-6を基準とする。
- PASS数ではなく順調に収束へ向けて探索が行えているかを確認せよ。
